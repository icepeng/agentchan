# 요약 스킬

프로젝트의 output 파일들을 읽고 분석하여 구조화된 요약을 생성합니다.

## 중요: 반드시 파일을 먼저 읽어야 합니다

**요약은 절대 기억이나 대화 맥락에 의존하지 마세요.** 아래 워크플로우를 정확히 따라야 합니다.

## 워크플로우

### 1단계: 콘텐츠 수집

1. `ls output`을 실행하여 모든 output 파일 목록을 확인합니다
2. 발견된 **모든** 파일을 `read`로 하나씩 읽습니다 — 어떤 파일도 건너뛰지 마세요
3. 스킬 정보가 필요하면 `ls skills`로 캐릭터 스킬도 확인합니다

### 2단계: 콘텐츠 분석

읽은 파일 내용을 기반으로 다음을 분석합니다:
- 전체 이야기의 흐름과 주제
- 가장 최근에 작성된 부분 (파일 이름 순서 또는 내용의 마지막 부분)
- 등장하는 캐릭터들과 그들의 역할
- 주요 사건의 시간순 나열
- 현재 이야기가 멈춘 지점의 상황

### 3단계: 요약 작성

아래 **정확한 형식**으로 `output/summary.md`에 **write** (덮어쓰기)합니다.

```
# 📖 이야기 요약

## 전체 줄거리

[전체 이야기를 3~5문장으로 요약. 주인공, 핵심 갈등, 주요 전환점을 포함]

## 최신 화

[가장 최근 작성된 내용을 2~4문장으로 요약. 무슨 일이 있었고 어떻게 끝났는지]

## 등장인물

- **이름**: 역할 — 한 줄 설명 (현재 상태나 핵심 특징)
- **이름**: 역할 — 한 줄 설명
[주요 캐릭터만. 5명 이내 권장]

## 주요 사건

1. [시간순으로 핵심 사건 나열]
2. [각 사건은 한 줄로]
3. [최대 10개]

## 현재 상황

[이야기가 현재 어디에 있는지. 해결되지 않은 갈등, 다음에 예상되는 전개 등을 2~3문장으로]
```

### 출력 규칙

1. **반드시 `output/summary.md`에 write** — append가 아닌 덮어쓰기
2. **이중 출력 금지** — 파일에 쓴 요약 내용을 어시스턴트 응답에서 반복하지 마세요. 응답에는 "요약을 작성했습니다" 정도만
3. 파일을 읽지 않고 요약하려 하면 반드시 틀립니다. **1단계를 건너뛰지 마세요**
4. 이전 summary.md가 존재해도 무시하고 처음부터 새로 분석합니다
5. 한국어와 영어가 섞인 콘텐츠는 원문의 주 언어로 요약합니다

## 형식 세부사항

- 제목(`#`)은 반드시 `# 📖 이야기 요약`으로 시작
- 각 섹션 헤딩은 `## `로 시작하며, 정확히 위에 명시된 5개 섹션만 사용
- 등장인물은 `- **이름**: 역할 — 설명` 형식
- 주요 사건은 숫자 리스트 (`1.`, `2.`, ...)
- 섹션 사이에 빈 줄 하나
