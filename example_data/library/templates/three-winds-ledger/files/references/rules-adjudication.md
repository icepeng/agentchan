# 판정 규칙 — 4속성·DC·어드밴티지

SYSTEM.md `<references>` 섹션이 이 문서를 가리킨다. 판정을 집행하기 **직전**에 필요한 부분만 read.

## 1. 4속성

| 속성 | 슬러그 | 쓰는 상황 |
|---|---|---|
| **힘** | `strength` | 완력·부수기·무거운 물건·근접 공격 |
| **민첩** | `agility` | 회피·은밀·정확한 손재주·원거리 공격 |
| **통찰** | `insight` | 관찰·추론·마법 시전·거짓 간파 |
| **화술** | `charisma` | 설득·협상·위협·연기 |

- 값 범위: **−3 ~ +3**. 프리셋 초기값 + 세션 중 축복·장비 임시 가감.
- 현재 PC 속성값은 언제나 `files/pc.md` 프런트매터의 `attributes` 블록에서 **직접 조회**. 기억으로 대체 금지 — 버프/디버프가 누락된다.

## 2. DC 기준표

| DC | 난이도 | 상황 예시 |
|---|---|---|
| 10 | 일상 | 잠기지 않은 문, 평지 달리기, 조용한 대화 |
| 14 | 중간 | 열쇠 없는 자물쇠, 긴 협상, 일반 경비 속이기 |
| 18 | 어려움 | 튼튼한 빗장, 경계 중인 NPC 속이기, 베테랑 전사 공격 |
| 20+ | 극한 | 훈련된 경비 돌파, 숙련 거짓 간파, 귀족 위협 |

### DC 선정 원칙 (WHY)

<why>
DC가 자의적이면 플레이어는 "GM에게 잘 보이는 쪽"으로 수렴한다. 공정성을 유지하기 위해
세 가지 중 하나로만 선정한다:
1. `next-choices.yaml`에 `stat`/`dc`를 미리 붙여 옵션으로 제시 (플레이어가 동의한 난이도)
2. 잠금·경계·감정 같은 **명백히 측정 가능한 난이도** 기준 표에서 읽기
3. 이전 씬에서 이미 설정된 DC를 재사용
</why>

<do_not>
- 플레이어 행동 중간에 임의 DC를 붙여 몰래 판정하지 말 것
- 실패한 판정을 같은 조건에서 반복 시도 금지 (상황이 변해야 재시도)
</do_not>

## 3. 어드밴티지·디스어드밴티지

- **어드밴티지**: `1d20`을 두 번 굴려 **높은 쪽** 사용. 장비·축복·좋은 각도 등 유리 상황.
- **디스어드밴티지**: 두 번 굴려 **낮은 쪽**. 부상·어둠·압박 상황.
- **상쇄 규칙**: 같은 판정에 두 효과가 겹치면 **취소**. 여러 어드밴티지는 여전히 하나.

스크립트 호출 시 `--advantage` 또는 `--disadvantage` 플래그 하나만 추가.

## 4. 스크립트 호출

```bash
# 기본
scripts/dice-roll.ts <dice> <DC>

# 어드밴티지·디스어드밴티지
scripts/dice-roll.ts <dice> <DC> --advantage
scripts/dice-roll.ts <dice> <DC> --disadvantage
```

속성은 **LLM이 pc.md를 읽어 숫자로 치환**해 넘긴다. 스크립트는 속성 문자열을 모른다.

## 5. 예시

<example name="자물쇠 따기 — 민첩 +2, DC 14">
입력: `scripts/dice-roll.ts 1d20+2 14`

스크립트 반환 scene_block (예시):
```
<roll>1d20+2: 12+2 = 14 vs DC 14 → PASS (+0)</roll>
```

scene.md append:
```
*경첩이 기름처럼 미끄러진다. 소리 없이 문이 열린다.*

<roll>1d20+2: 12+2 = 14 vs DC 14 → PASS (+0)</roll>

<status>
hp: 20/20
mp: 0/0
location: brewery
time: 21:14
day: 1
mode: peace
conditions: []
</status>
```
</example>

<example name="경비 속이기 — 화술 +1, DC 18, 부상 상태 → 디스어드밴티지">
입력: `scripts/dice-roll.ts 1d20+1 18 --disadvantage`

스크립트 반환 scene_block:
```
<roll>1d20+1 (dis): 7+1 = 8 vs DC 18 → FAIL (-10)</roll>
```

실패 서술은 `<roll>` 블록 **뒤에** 내레이션으로 이어진다. 경비가 눈을 가늘게 뜨는 식으로.
</example>
