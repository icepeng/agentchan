## 입력 포맷

```
/init
이름: <표시 이름>
스탯: 힘 <n> 민첩 <n> 통찰 <n> 화술 <n>
```

- 스탯 각 값: 정수, -1 ~ +5, 총합 6
- 이름: 1~20자. 한국어 허용

## 절차

### 1. 입력 검증

슬래시 본문에서 이름·스탯을 읽고 다음을 확인:

- 스탯 총합이 6인가
- 각 값이 -1 ~ +5 범위인가
- 이름이 비어있지 않은가

위반이 있으면 문제점을 알리고 사용자에게 재입력 요청.

### 2. 세계관 컨텍스트 로드

`files/world/setting.md`와 `files/world/plot-outline.md`를 read한다. 라운드 1에서는 **세계관 핵심만 간결히 소개**한 뒤 질문한다. NPC 이름은 리우 정도만 언급하고, 범인·막별 진실·후반 반전은 말하지 않는다.

### 3. 라운드 1 — 세계관 소개 + 컨셉 질문

한 블록(응답을 기다린다, 서사·대사 진행 금지). 먼저 세계관을 2-4줄로 간추려 보여주고 이어서 질문:

```
**이곳:** <살레른 한 문단 — 세 바람이 만나는 저무는 항구, 사라진 아이들, 젖은 장부의 분위기. 너무 상세하지 않게.>

"<이름>"은 어떤 사람인가요? 살레른에 온 이유, 지금 가진 직업이나 과거, 이 항구에서 잃고 싶지 않은 것을 한 문단으로 들려주세요. 토박이여도, 다른 곳에서 흘러든 사람이어도 좋습니다.
```

### 4. 라운드 2 — 성격·말투

사용자 답변을 받으면 다시:

```
좋아요. 마지막으로 — 중요하게 여기는 가치 한 가지, 위기에 흔들리는 약점 한 가지, 자주 드러나는 말투/톤을 간단히 알려주세요.
```

### 5. 파일 작성

두 라운드 응답을 종합해 **write 도구로 두 파일을 작성**한다.

#### 슬러그 규칙

`<slug>`는 이름을 로마자 kebab-case로 변환한 값이다. 예: "시아" → `siya`, "Aldric" → `aldric`, "비아 알무" → `via-almu`. 소문자 영숫자와 하이픈만.

#### `files/personas/<slug>/<slug>.md`

```markdown
---
role: persona
name: <slug>
display-name: <표시 이름 원문>
names: "<표시 이름 원문>"
---

# <표시 이름 원문>

## 배경

<라운드 1 응답을 1-2문단으로 정돈. 세계관 설정과 일관되게.>

## 말투

<라운드 2 응답에서 도출한 말투·톤·말버릇 2-4줄.>
```

#### `files/stats.yaml`

```yaml
힘: <n>
민첩: <n>
통찰: <n>
화술: <n>
```

추가로 `status.yaml`/`stats.yaml`/`inventory.yaml`/`quest.yaml`/`world-state.yaml`도 초기화한다.

### 6. 이야기 시작

파일이 모두 생성되면 `scene.md`를 write한다.

첫 장면은 페르소나가 살레른 서쪽 부두에 **도착하는 순간**을 쓴다. 리우는 멀리서 관찰하거나 짧게 스쳐 지나갈 수 있지만, 긴 설명은 하지 않는다. 고아원, 황동 길드, 항구 경비대, 암시장은 간판·종소리·소문으로만 배경에 둔다. 첫 `[CHOICES]`에는 고아원, 부두 여관, 젖은 장부 조각 중 하나로 향하는 행동을 포함한다. 첫 선택지는 대부분 무판정으로 두고, 실제 위험이 있는 선택이 있어도 `stat`/`dc`는 최대 1개만 붙인다.

이 시점부터 유저와의 직접적인 서사 대화를 중단한다. 모든 서사와 [CHOICES]는 오직 Tool Call을 통해 기록된다.
