# ending-scene

수사는 무한히 이어지지 않는다. 아래 트리거 중 하나가 만족되면 활성화해 엔딩 씬을 기록한다.

<when-to-use>

엔딩 트리거 — 하나라도 만족되면 활성화:

- **(T0) 범인의 자백 발화 — 최우선·즉시 활성화**. 범인 NPC가 scene.md에 **방어를 포기하고 범행을 인정하는 발화**를 한 턴. 단서 수·신뢰 수치와 **무관**. 자백 = 즉시 엔딩 모드 전환.

  **자백 시그니처 예시** (포함되면 자백으로 간주):
  - `"그래요, 제가 지웠어요."`
  - `"맞아요. 제가 한 거예요."`
  - `"…미안해."` (첫 존댓말 탈락 + 사과)
  - `"역시 선배는 못 속여."` (탄식형 인정)
  - `"더 숨길 필요 없겠네요."` (방어 포기 선언)

- **(A) 공식 지목** — 유저가 **세 NPC 앞에서 특정 한 명을 범인으로 명시적 지목**한 턴. 범인이 자백하든 방어하든 **지목 발화 턴 끝에** 활성화. E1~E3 분기는 이 스킬 내에서 현재 상태로 판정한다 (지목 대상이 실제 범인인지, 단서 몇 개 해금됐는지, 범인 신뢰 수치가 얼마인지).
- **(B) 유도 자백** — 범인 NPC가 +3 이상이거나 결정적 단서 2개 이상 해금된 상태에서 유저가 단독 대화로 몰아붙임
- **(C) 세션 종료 선언** — 유저가 OOC로 "오늘은 여기까지", "마무리", `[OOC: ...]` 등을 명시
- **(D) 시간 임계(선택)** — `scene.md`의 씬 수가 8개 이상일 때, "새벽 1시가 가까워진다" 같은 자연스러운 시간 압박으로 결단을 유도 (강제 금지)

</when-to-use>

<activation-sequence>

자백 시그니처(T0) 또는 공식 지목(A)이 발생한 턴의 **처리 순서**:

1. `append(scenes/scene.md)` — 유저 메시지 에코 + 본 턴의 NPC 반응·자백 대사·지문
2. 큐 이벤트가 있으면 `write(stats.md)` — 변화 수치 반영
3. **같은 턴 안에서 곧바로** `activate_skill(ending-scene)` 호출
4. 스킬 body를 받은 뒤, `append(scenes/scene.md)` — `---` 구분선 + 엔딩 본문 + `---` + 에필로그
5. 어시스턴트 응답에 재시작 안내 한 줄

**금지**: 자백이 나온 턴 끝에 엔딩을 미루고 다음 턴으로 넘기지 말 것. "동기를 더 물어보시겠습니까?" 같은 진행 프롬프트는 자백 이후엔 **무효** — 엔딩이 이미 시작되어야 한다.

</activation-sequence>

<endings>

`suspects-truth.md`의 `ending.*` 기준값과 현재 상태(해금 단서 수·범인 신뢰·유저가 누구를 지목했는가)를 비교해 **하나만** 고른다.

### E1. 완전 해결 (Resolved)
- **조건**: 범인 정답 + 핵심 단서 **3개 이상 해금** + (범인 NPC **+3 이상** 또는 **결정적 단서로 몰아붙이기 성공**)
- **씬**: 범인이 무너지며 자백. **표면 동기 뒤의 진짜 그림(반전)이 이때 공개**. 무고한 둘은 각자의 시그니처 동작으로 반응 (민지 메모장 덮기, 이서 화면 끄기 등)
- **톤**: 슬프지만 명료한 해결. 유저는 파일 복구 약속과 함께 자리를 뜬다

### E2. 부분 해결 (Partial)
- **조건**: 범인 정답 + 단서 2개 해금, 하지만 반전 미공개
- **씬**: 범인은 "맞아요"까지만 말하고 동기는 침묵. 유저가 떠난 뒤 남은 둘이 더 이야기할 여운
- **톤**: 수사는 끝났지만 상처는 남음. 뒤를 누가 책임질지 미결

### E3. 오판 (Misjudgment)
- **조건**: 유저가 엉뚱한 NPC를 지목, 단서 2개 미만 해금
- **씬**: 지목당한 무고한 NPC가 침묵하거나 방어적으로 긍정 ("그게 맞다면 맞는 거겠죠"). 진범은 안도를 숨김 — **`cracked` 삽화 금지, `calm`으로 냉각**
- **톤**: 씁쓸한 어둠. 에필로그에서 진실이 아주 희미하게 암시됨 ("형광등이 다시 깜빡였다" 수준. 진범 이름을 직접 쓰지 말 것)

### E4. 열린 결말 (Open)
- **조건**: (C) 종료 선언 또는 (D) 시간 임계 트리거, 유저가 지목 없이 세션 종료
- **씬**: 수사는 미완. 새벽으로 넘어가는 시간, 세 NPC와 유저가 각자 자리에 남는다
- **톤**: 다음 세션 여지

</endings>

<scene-rules>

1. **엔딩 씬은 `---` 구분선으로 시작** — 이전 수사 씬과 분리
2. **에필로그 1장면** — 엔딩 본문 뒤에 `---` 하나 더, 시간 점프 후 짧은 장면 (2~5줄). 서술 톤이 가장 강하게 드러나는 마지막 모먼트
3. **`stats.md` 리셋 금지** — 수치를 0으로 되돌리지 않는다. 세션의 감정 상태가 그대로 남게
4. **재시작 안내** — 어시스턴트 응답에 **한 줄**: `"세션을 다시 시작하려면 files/suspects-truth.md를 삭제하고 scene.md를 비우세요."` 이 안내는 엔딩 씬 append **직후에만** 출력

</scene-rules>

<npc-reactions>

| NPC | 본인이 범인 (E1) | 무고·정답 엔딩 (E1) | 오판에 지목당함 (E3) |
|---|---|---|---|
| 이서 | 화면 끄고 안경 벗음. 드문 침묵 길이 | 짧은 팩트 한 줄로 사건 정리 | "로그가 맞다면." — 부정도 긍정도 아님 |
| 한결 | 팔짱 풀고 "…미안해" — 첫 존댓말 탈락 | 책상 치기 대신 머리카락 쓸기 | 폭발 직전에서 멈춤. 문 쪽 응시 |
| 민지 | 메모장 덮고 소매 끝 끝까지 말아 쥠 | 세 사람 자리를 둘러봄 — 가장 길게 남음 | 고개 들지 못하고 "저는…"으로 끊김 |

</npc-reactions>

<example>

E1 씬 한 토막 (범인=한결, 반전=hidden_motive: 코드가 한결 과거 작품과 동일):

```
---

[hangyeol:assets/cracked] **한결:** *팔짱을 풀고 책상 위에 양손을 올린다. 머리카락을 한 번 쓸어넘긴다.* "…미안해."

*이서는 화면을 끈다. 키보드 위에 얹었던 손이 천천히 내려온다. 민지는 메모장을 덮고 소매 끝을 주먹으로 끝까지 말아 쥔다.*

**한결:** *시선이 떨어진다.* "그 파일 안의 코드, 2년 전 내가 과제로 낸 거랑 똑같아요. 교수님 이름으로 저널에 나간 논문 샘플이랑도. 증거가 남으면… 누가 누구 걸 베꼈는지 따지게 될 거라."

> (유저) "그래서 지웠어?"

**한결:** "지웠어요. 복구는 가능하게 남겼어요. 대회는 망치기 싫었거든요."

*형광등이 한 번 더 깜빡였다. 민지가 고개를 아주 조금 든다.*

*→ 한결 -3*

---

*30분 뒤. 선배는 동아리실 문 앞에 선다. 백업 드라이브가 배낭 안에서 무게를 남긴다. 이서는 화면을 다시 켜지 않았다. 민지는 한결 옆자리로 의자를 끌고 갔다. 한결은 고개를 들지 않는다. 문이 닫힌다.*
```

에필로그 뒤에 어시스턴트 응답 한 줄:
> 세션을 다시 시작하려면 files/suspects-truth.md를 삭제하고 scene.md를 비우세요.

</example>
