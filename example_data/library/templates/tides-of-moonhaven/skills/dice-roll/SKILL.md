# Dice Roll

RPG 판정에서 **실제 난수(crypto RNG)**를 사용하는 주사위 스킬입니다. 직접 숫자를 만들지 말고 반드시 `script` 도구로 `roll.ts`를 실행하세요.

## 사용법

```
script(file: "skills/dice-roll/scripts/roll.ts", args: ["<dice>", "<DC>"])
```

| 표기 | 의미 |
|------|------|
| `1d20` | 20면체 1개 |
| `1d20+3` | 20면체 1개 + 보정 +3 |
| `2d6+3` | 6면체 2개 + 3 |
| `1d20-2` | 20면체 1개 - 2 |
| `d100` | = 1d100 |
| `4d6kh3` | 6면체 4개 중 상위 3개만 합산 |

## 판정 예시 흐름

상황: 사용자가 "자물쇠를 따본다" (도적 페르소나, 민첩 +3, DC 12 민첩 판정)

1. `stats.yaml` 조회 → `민첩: 3`
2. script 호출: `args: ["1d20+3", "12"]`
3. 결과:
   ```
   Notation: 1d20+3
   Roll: 8
   Modifier: +3
   Total: 8 +3 = 11
   DC 12: FAIL (margin -1)
   ```
4. 서사 반영 — 간발의 차로 실패, 핀이 미끄러진다
5. 기록: `[SYSTEM] 판정: 민첩 DC 12 → 실패 (주사위 8 +민첩 3 = 11)`

보정이 0이면 `1d20`로 호출하고 기록도 `(주사위 N)`만. `[CHOICES]`에 `stat: <능력치>`가 있으면 `stats.yaml`에서 그 값을 찾아 주사위에 반영합니다.

## 판정을 언제 거는가

- 불확실한 행동, 위험, 정보 획득 — 드라마틱한 순간에만. 일상 행동은 판정 없이
- 선택지마다 판정을 붙이지 않는다. 실패가 장면을 바꾸지 않는 행동은 그냥 성공시키거나 대화로 이어간다
- 상황의 유·불리는 **DC**로 조정. 스탯 보정치는 즉흥 변경 금지
- DC는 10-15에 몰리지 않게 아래 기준에서 고른다:
  - 6-8: 낮은 위험. 조용히 살피기, 쉬운 단서, 가벼운 균형 잡기
  - 10-12: 평범한 장애물. 보통 자물쇠, 낯선 상대 설득, 흐린 흔적 찾기
  - 15: 실패 대가가 분명한 행동. 관계 악화, 피해, 단서 지연
  - 18: 명백히 불리한 조건. 경계 중인 상대, 폭풍, 복잡한 장치
  - 22: 거의 불가능한 도박. 중요한 장면에서만 드물게
- script가 반환한 결과는 그대로 사용 — 재굴림·무시 금지. 실패는 서사적 전환점
