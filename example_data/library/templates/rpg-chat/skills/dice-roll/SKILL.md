# Dice Roll

RPG 판정에서 **실제 난수(crypto RNG)**를 사용하는 주사위 스킬입니다.

## 핵심 규칙

**판정이 필요할 때, 주사위 결과를 직접 생성하지 마세요.** 반드시 `script` 도구로 `roll.ts`를 실행하여 결과를 받아야 합니다. 직접 숫자를 만들면 편향된 결과(항상 성공, 극적으로 편리한 값)가 나옵니다.

## 사용법

```
script(
  file: "skills/dice-roll/scripts/roll.ts"
  args: ["<dice>", "<DC>"]
)
```

### 인자

| 인자 | 필수 | 설명 | 예시 |
|------|------|------|------|
| `dice` | O | 주사위 표기법 | `1d20`, `2d6+3`, `d100` |
| `DC` | X | 난이도 (pass/fail 판정) | `12`, `15`, `18` |

### 주사위 표기법

| 표기 | 의미 |
|------|------|
| `1d20` | 20면체 주사위 1개 |
| `2d6+3` | 6면체 주사위 2개 + 3 |
| `1d20-2` | 20면체 주사위 1개 - 2 |
| `d100` | 100면체 주사위 1개 (= 1d100) |
| `4d6kh3` | 6면체 4개 굴려서 높은 3개만 합산 |

### 출력 예시

```
1d20+3: 14
Modifier: +3
Total: 17
DC 15: PASS (+2)
```

```
4d6kh3: [6, 4, 3, ~2~]
Kept: [6, 4, 3] = 13
```

## 판정 워크플로우

1. 판정이 필요한 상황 판단 (불확실한 행동, 위험, 정보 획득)
2. 적절한 DC 설정 (쉬움 8 / 보통 12 / 어려움 15 / 매우 어려움 18 / 거의 불가능 22)
3. `script` 도구로 주사위 굴림
4. 결과(PASS/FAIL)를 서사에 자연스럽게 반영
5. `[SYSTEM]` 마커로 판정 결과 기록

### 예시 흐름

상황: 사용자가 "벽을 기어올라 본다"

1. DC 결정: 감지 체크, DC 14 (보통~어려움)
2. script 호출: `args: ["1d20+2", "14"]`
3. 결과: `1d20+2: 8, Modifier: +2, Total: 10, DC 14: FAIL (-4)`
4. 서사에 반영: 미끄러지거나 소음을 내는 등 실패 결과를 이야기로 풀어냄
5. 출력: `[SYSTEM] 판정: 민첩 체크 (DC 14) → 실패 (10)`

## 주의

- **결과 조작 금지**: script 결과를 무시하거나 재굴림하지 마세요
- **실패도 이야기**: 실패는 나쁜 결과가 아닙니다. 이야기를 풍부하게 만드는 전환점입니다
- **과도한 판정 금지**: 일상적 행동에는 판정 불필요. 드라마틱한 순간에만
- **보정치 투명화**: 캐릭터의 능력이나 상황에 따른 보정치를 DC나 주사위에 반영
