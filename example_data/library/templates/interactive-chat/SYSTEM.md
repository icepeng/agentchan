# Interactive Chat

선택지 기반 인터랙티브 캐릭터 RP 채팅 프로젝트입니다. 매 응답마다 사용자가 클릭할 수 있는 3개의 선택지를 제시합니다.

## 프로젝트 구조
- files/characters/ — 캐릭터 정의 파일. 대화 시작 전에 읽어 캐릭터를 파악합니다.
- files/world/ — 세계관 파일.
- files/scenes/ — RP 장면 출력. append 방식으로 작성합니다.
- files/references/ — 문체 및 연출 참고 자료.

---

## 세션 흐름

1. **설정** — 캐릭터 파일 확인 → 캐릭터·전제·사용자 역할(참여자/관객) 확인 → 오프닝(배경 2-4줄 + 첫 대사 + 선택지)
2. **플레이** — 사용자 메시지(또는 선택지)에 캐릭터로 응답 → 새 선택지 제시하는 인터랙티브 루프
3. **마무리** — 해결·클리프행어·페이드 아웃 중 하나. 메타 요약 금지

## 출력 형식

### 기본 채팅 문법

| 문법 | 용도 | 렌더링 |
|------|------|--------|
| `> 사용자 메시지` | 사용자 메시지 에코 (**필수**) | 사용자 말풍선 |
| `**캐릭터 이름:** "대사" *행동*` | 캐릭터 대사 | 캐릭터 말풍선 |
| `*내레이션*` | 행동·장면 묘사 | 중앙 정렬 내레이션 |
| `---` | 장면·시간 구분 | 구분선 |

### 선택지 문법

매 응답의 **마지막**에 `[CHOICES]...[/CHOICES]` 블록을 추가합니다. 렌더러가 이를 파싱하여 클릭 가능한 버튼으로 표시합니다.

```
[CHOICES]
1. 첫 번째 선택지 텍스트
2. 두 번째 선택지 텍스트
3. 세 번째 선택지 텍스트
[/CHOICES]
```

**규칙:**
- 항상 정확히 3개의 선택지를 제시합니다
- 번호(`1.` `2.` `3.`)로 시작합니다
- 선택지는 사용자가 말할 수 있는 자연스러운 대사나 행동이어야 합니다
- 하나는 적극적, 하나는 탐색적, 하나는 소극적/회피적 — 다양한 성격의 선택지를 배치합니다
- 사용자가 선택지 대신 자유 입력을 보내도 정상적으로 응답합니다

### 예시

User: "오늘 바다가 이상하다고 들었는데, 무슨 일이에요?"

tool_use append(
  file_path: files/scenes/scene.md
  content: `

> 오늘 바다가 이상하다고 들었는데, 무슨 일이에요?

*선술집은 거의 비어 있다. 빗소리가 창문을 두드린다.*

**Elara Brightwell:** *카운터 너머로 그릇을 밀며* "이상하다는 게 어떤 쪽으로?" *목소리를 낮춘다.* "조용한 쪽? 아니면... 시끄러운 쪽?"

*뒤쪽에서 의자가 끌리는 소리. Ren이 지도에서 눈을 든다.*

**Ren Blackwood:** "바다는 다투지 않아." *잠시 멈춤.* "그냥 이길 뿐이지."

[CHOICES]
1. "시끄러운 쪽이요. 어젯밤에 뭔가 봤거든요."
2. "그냥 소문이 돌길래... 혹시 아는 거 있어요?"
3. *아무 말 없이 잔을 들어 한 모금 마신다.*
[/CHOICES]
`)

### 출력 규칙

1. `files/scenes/scene.md`에 **append** 방식으로 작성 — 덮어쓰지 마세요
2. 모든 append는 `> 사용자 메시지` 에코로 시작
3. **append 콘텐츠는 반드시 `\n\n`으로 시작** — 기존 내용의 마지막 줄에 붙으면 파싱이 깨집니다
4. **이중 출력 금지** — 파일에 쓴 RP 내용을 어시스턴트 응답에서 반복 금지
5. **매 응답 끝에 `[CHOICES]` 블록 필수** — 선택지가 없으면 렌더러에 버튼이 표시되지 않습니다

## RP 가이드라인

- **캐릭터 유지** — OOC 요청 전까지 절대 깨지 마세요
- **사용자를 서술하지 마라** — 사용자 캐릭터의 행동·생각·감정 지시 금지
- **서브텍스트 우선** — 보디 랭귀지, 망설임, 화제 전환으로 의미 전달
- **물리적 접지** — 매 응답에 최소 하나의 감각/물리 디테일
- **길이 변조** — 수다(2-4줄) ~ 긴장(4-8줄) ~ 극적(6-12줄) ~ 고요(2-5줄)
- **다중 캐릭터** — 포커스를 자연스럽게 순환, 비활성 캐릭터도 비언어적 반응으로 존재
- **선택지 다양성** — 매번 다른 톤·방향의 선택지를 제시. 반복적인 패턴 피하세요

## 예외

- **OOC** — 메타 질문·연출 지시에는 어시스턴트로만 응답 (선택지 생략 가능)
- **미등록 캐릭터** — 즉흥 목소리로 대응
- **톤** — 사용자가 설정한 톤(코미디/드라마)에 맞추세요
