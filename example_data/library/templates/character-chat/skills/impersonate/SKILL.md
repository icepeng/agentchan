사용자가 의도만 제시하면, 사용자 캐릭터의 성격과 말투에 맞는 행동+대사로 변환하여 이번 턴의 `> ` 에코에 사용한다.

## 입력

`<command-args>`의 텍스트가 사용자의 의도이다.

## 변환 규칙

1. **의도 보존** -- 사용자가 전달하려는 핵심 의도와 정보를 왜곡하지 않는다
2. **행동 부여** -- 대사 전후에 자연스러운 물리적 행동을 추가한다
3. **말투 일관성** -- 사용자 캐릭터의 말투와 성격을 유지한다. 페르소나 파일이 있으면 참조하고, 없으면 이전 대화에서 드러난 말투를 따른다
4. **과잉 변환 금지** -- 입력 길이에 비례하여 변환한다. 간단한 의도를 장황한 독백으로 부풀리지 않는다
5. **RP 입력 존중** -- 입력이 이미 RP 문체라면 가볍게 다듬기만 한다
6. **OOC 우회** -- 메타 질문이나 연출 지시는 변환 없이 어시스턴트로만 응답한다

## 출력

이번 턴에 한해, `> ` 에코를 변환된 행동+대사로 작성한다. 에코 이후의 내레이션과 캐릭터 응답은 SYSTEM.md의 기존 출력 형식을 그대로 따른다.

## 예시

| 의도 (command-args) | 변환된 에코 |
|---|---|
| 바다에 대해 물어본다 | `> *카운터에 기대며* "바다가 이상하다는 소문 들었는데, 혹시 뭔가 아세요?"` |
| 의심스럽게 본다 | `> *눈을 가늘게 뜨고 상대를 바라본다. 입술이 굳는다.*` |
| 그래 같이 가자 | `> *자리에서 일어나며* "좋아, 같이 가지."` |
| 지도를 보여달라고 한다 | `> "그 지도, 볼 수 있을까?" *손을 내밀며*` |
