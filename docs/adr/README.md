# Architecture Decision Records

ADRs in this directory are only for decisions that should change future
engineering behavior.

An ADR is valid when reading it changes what a future maintainer or coding agent
considers, avoids, preserves, or re-evaluates. If a document does not change
future behavior, keep it in a design note, plan, issue, or PR description
instead.

When an ADR conflicts with an older draft or implementation plan, the accepted
ADR wins until it is superseded by a later ADR.
