## 편집 모드란?

편집 모드는 파일 트리와 CodeMirror 에디터로 프로젝트의 모든 파일을 직접 편집할 수 있는 모드입니다. 에이전트에게 지시하지 않고도 파일 내용을 즉시 확인하고 수정할 수 있습니다.

## 진입 방법

에이전트 패널(오른쪽) 헤더의 토글 버튼으로 Chat과 Edit 모드를 전환합니다. Chat 모드에서는 에이전트와 대화하고, Edit 모드에서는 파일을 직접 편집합니다.

## 편집 가능한 파일

편집 모드에서 수정할 수 있는 파일은 다음과 같습니다.

- **`SYSTEM.md`** — 시스템 프롬프트를 미세 조정하여 에이전트의 행동 방식을 변경합니다.
- **`skills/*/SKILL.md`** — 프로젝트에 등록된 스킬의 내용을 수정합니다.
- **`renderer.ts`** — 렌더러를 커스터마이징하여 시각적 출력 방식을 변경합니다.
- **`files/**`** — 캐릭터, 세계관, 장면 등 모든 작업 파일을 편집합니다.

## 파일 트리

편집 모드에 진입하면 왼쪽에 프로젝트의 파일 트리가 표시됩니다. 파일을 클릭하면 오른쪽 에디터 영역에서 해당 파일이 열립니다.

## 컨텍스트 메뉴

파일 트리에서 파일을 우클릭하면 컨텍스트 메뉴가 나타납니다.

- **삭제** — 선택한 파일을 삭제합니다.
- **탐색기에서 열기** — 해당 파일을 시스템 파일 탐색기에서 엽니다.

## 미저장 변경

파일을 수정한 후 저장하지 않은 상태에서 다른 파일로 전환하거나 모드를 전환하면 확인 다이얼로그가 표시됩니다. 변경사항을 저장하거나 취소한 후 이동할 수 있습니다.

## 활용 사례

- **SYSTEM.md를 직접 수정**하여 에이전트의 행동을 미세 조정합니다. 예를 들어, 에이전트의 문체나 규칙을 세부적으로 변경할 때 유용합니다.
- **renderer.ts를 편집**하여 시각적 출력을 커스터마이징합니다. 레이아웃, 스타일, 표시 로직 등을 직접 제어할 수 있습니다.
- **캐릭터 파일의 설정값이나 설명을 직접 수정**합니다. 에이전트에게 지시하는 것보다 빠르게 세부 사항을 변경할 때 편리합니다.
