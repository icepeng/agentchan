## SYSTEM.md란?

SYSTEM.md는 프로젝트 행동의 **단일 원천**입니다. Plain markdown 파일로 작성하며, 그 내용이 시스템 프롬프트에 원문 그대로 주입됩니다. 에이전트가 "어떤 존재이고 어떻게 행동할지"를 이 파일 하나로 정의합니다.

## 시스템 프롬프트 구성

에이전트가 받는 시스템 프롬프트는 세 부분으로 구성됩니다.

```
[1] 기본 시스템 프롬프트 (하드코딩 — 도구 사용 규칙)
[2] SYSTEM.md 본문 (원문 그대로)
[3] 스킬 카탈로그 (이름+설명 자동 생성)
```

기본 시스템 프롬프트는 도구 사용 규칙 등 공통 지시를 포함하며, 그 뒤에 SYSTEM.md 본문이 그대로 이어집니다. 마지막으로 프로젝트에 등록된 스킬들의 카탈로그(이름과 설명)가 자동으로 추가됩니다.

## 작성 가이드

### 프로젝트 구조 정의

`files/` 디렉토리의 예상 구조를 명시하세요. 에이전트가 어디에 무엇을 저장할지 안내합니다.

```markdown
## 파일 구조

- files/characters/{name}/ — 캐릭터 정의 파일
- files/characters/{name}/assets/ — 캐릭터 이미지
- files/scenes/ — 장면 출력
- files/world/ — 세계관 설정
```

### 세션 흐름

에이전트가 따라야 할 워크플로우를 정의합니다.

```markdown
## 세션 시작

1. files/characters/를 읽고 등장인물을 파악한다
2. files/scenes/에서 마지막 장면을 확인하여 이어서 작성한다
3. 사용자의 입력을 기다린다
```

### 출력 형식

렌더러가 파싱할 수 있는 마크업 규칙을 정의합니다. 에이전트의 출력과 렌더러의 파싱 로직이 일치해야 시각적 출력이 올바르게 표시됩니다.

```markdown
## 출력 형식

- 사용자 발화: `> user message`
- 캐릭터 대사: `**Character:** "dialogue" *action*`
- 나레이션: 일반 텍스트
```

### RP/작업 가이드라인

캐릭터 유지 규칙, 문체 지침, 금지 사항 등 에이전트의 행동 규칙을 작성합니다.

```markdown
## 규칙

- 캐릭터의 성격과 말투를 일관되게 유지할 것
- 사용자 캐릭터를 대신 행동하지 말 것
- 한국어로 응답할 것
```

### 세션 이어가기

대화가 길어지면 컴팩션(요약)이 발생합니다. 컴팩션 후에도 에이전트가 맥락을 복원할 수 있도록 방법을 기술합니다.

```markdown
## 세션 이어가기

새 세션 시작 시 또는 요약 이후:
1. files/scenes/에서 최근 장면을 읽어 맥락을 복원한다
2. 캐릭터 파일을 다시 읽어 성격/상태를 확인한다
```

## 팁

- **파일 경로와 구조를 구체적으로 명시할 것** — 에이전트는 명시된 대로 파일을 다룹니다. 모호한 지시는 예상치 못한 결과를 낳습니다.
- **출력 형식을 명확히 정의할 것** — 렌더러가 이 형식을 파싱해야 합니다. 에이전트의 출력 형식과 렌더러의 파싱 규칙이 일치하지 않으면 화면이 깨집니다.
- **Markdown으로 작성** — SYSTEM.md는 plain markdown이므로 컴팩션에 안전합니다. 특수한 문법이나 포맷이 필요 없습니다.
- **Edit Mode에서 직접 편집 가능** — 에이전트 패널 상단의 Edit Mode 토글을 켜면 SYSTEM.md를 직접 편집할 수 있습니다.
