## SKILL.md 형식

각 스킬은 `SKILL.md` 파일로 정의됩니다. YAML frontmatter에 메타데이터를, 본문에 에이전트 지시사항을 작성합니다.

```markdown
---
name: skill-name
description: "스킬이 하는 일 한 줄 설명"
---

# 스킬 제목

스킬 본문 — 에이전트가 이 스킬을 활성화했을 때 읽게 되는 지시사항.
구체적인 단계와 규칙을 작성합니다.
```

## Frontmatter 필드

- **name** (필수): 스킬의 고유 이름. 슬래시 커맨드에서 `/name`으로 사용됩니다.
- **description** (필수): 스킬이 하는 일을 한 줄로 설명합니다. 이 설명이 시스템 프롬프트의 카탈로그에 표시되며, 에이전트가 스킬 선택 판단에 사용합니다.

## 스킬 디렉토리 구조

```
skills/{skill-name}/
├── SKILL.md        # 스킬 정의 (frontmatter + 본문)
├── scripts/        # 실행 가능한 TypeScript (script 도구로 실행)
└── assets/         # 템플릿, 데이터 등 보조 파일
```

- `SKILL.md`는 필수입니다. 나머지 디렉토리는 필요할 때만 생성합니다.
- `scripts/`에는 에이전트가 실행할 수 있는 TypeScript 파일을 둡니다.
- `assets/`에는 스킬이 참조하는 보조 파일(템플릿, 데이터 등)을 둡니다.

## 스크립트

`scripts/*.ts` 파일은 에이전트가 `script` 도구로 실행합니다. 스크립트 작성 시 다음 규칙을 따릅니다.

- **각 스크립트는 self-contained** — 외부 패키지나 다른 파일에 의존하지 않습니다. 필요한 로직은 모두 스크립트 파일 안에 포함합니다.
- **스킬 간 헬퍼 공통화 금지** — 여러 스킬에서 비슷한 로직이 필요하더라도 각 스크립트에 개별적으로 작성합니다. 소량의 중복은 허용됩니다.

## 작성 팁

- **스킬은 독립적이어야 합니다** — 다른 스킬에 의존하지 않도록 작성하세요. 각 스킬은 단독으로 활성화되어 동작해야 합니다.
- **description은 명확하게** — 에이전트가 카탈로그의 description을 보고 어떤 스킬을 활성화할지 판단합니다. 모호한 설명은 잘못된 스킬 선택으로 이어집니다.
- **본문에는 구체적인 절차를 단계별로 작성** — 에이전트가 스킬을 활성화하면 본문의 지시를 따릅니다. 명확한 단계와 규칙을 제공할수록 결과가 좋습니다.
- **Edit Mode에서 직접 편집 가능** — 에이전트 패널 상단의 Edit Mode 토글을 켜면 스킬 파일을 직접 편집할 수 있습니다.
