## 대화 세션

프로젝트별로 대화 기록이 세션 단위로 저장됩니다. 각 세션은 JSONL 형식의 파일로 관리되며, 헤더 라인과 메시지 트리 노드로 구성됩니다.

하나의 프로젝트에 여러 세션이 존재할 수 있으며, 세션 간 전환을 통해 다른 대화 흐름을 이어갈 수 있습니다.

## 긴 대화 관리

대화가 길어지면 모델의 컨텍스트 윈도우 한계에 가까워집니다. 컨텍스트 윈도우를 초과하면 이전 메시지가 잘리거나 에이전트의 응답 품질이 저하될 수 있습니다.

이 문제를 해결하기 위해 agentchan은 컴팩션(Compaction) 기능을 제공합니다.

## 컴팩션(Compaction)

컴팩션은 긴 대화를 자동으로 요약하여 새 세션으로 이어가는 기능입니다.

컴팩션이 실행되면 다음과 같은 과정이 진행됩니다:

1. **최근 메시지 유지** — 토큰 예산 기반 retention으로 최근 메시지는 그대로 보존합니다
2. **오래된 메시지 요약** — 보존 범위 밖의 오래된 메시지를 요약 노드로 압축합니다
3. **새 세션 생성** — 요약 노드와 최근 메시지를 포함한 새 세션이 만들어집니다

요약된 내용은 `compact-summary` 메타 노드로 새 세션의 시작 부분에 배치되어, 에이전트가 이전 대화의 맥락을 파악할 수 있습니다.

## 세션 연속성

대부분의 템플릿에는 SYSTEM.md에 "세션 이어가기" 섹션이 포함되어 있습니다. 이 지시사항과 컴팩션의 요약 노드가 연동되어, 컴팩션 이후에도 에이전트가 이전 맥락을 파악하고 자연스럽게 대화를 이어갈 수 있습니다.

예를 들어, 캐릭터 RP 세션에서 컴팩션이 일어나면 이전 장면의 요약이 새 세션에 포함되고, 에이전트는 그 요약을 바탕으로 이야기를 계속 진행합니다.
