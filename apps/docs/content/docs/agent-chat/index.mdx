## 대화 방식

화면 하단의 입력창에 메시지를 입력하면 AI 에이전트가 SSE(Server-Sent Events) 스트리밍으로 실시간 응답합니다. 응답이 완료되면 왼쪽 패널의 렌더러 출력이 자동으로 갱신됩니다.

에이전트는 단순히 텍스트를 생성하는 것이 아니라, 프로젝트의 파일을 직접 읽고 수정할 수 있는 도구를 갖추고 있습니다.

## 에이전트 도구

에이전트가 사용할 수 있는 도구 목록입니다. 모든 도구는 프로젝트 디렉토리 내에서만 동작하며, 쉘(bash, cmd 등)은 사용할 수 없습니다.

- **`read`** — 파일 내용 읽기
- **`write`** — 파일 생성 또는 덮어쓰기
- **`append`** — 기존 파일에 내용 추가
- **`edit`** — 기존 파일의 특정 부분 수정
- **`ls`** — 디렉토리 목록 확인
- **`grep`** — 파일 내용을 패턴으로 검색
- **`script`** — 스킬에 포함된 TypeScript 헬퍼 스크립트 실행

에이전트는 대화 중 필요에 따라 이 도구들을 자동으로 선택하여 사용합니다. 예를 들어, 새로운 캐릭터를 만들어 달라고 하면 `write` 도구로 캐릭터 파일을 생성하고, 기존 캐릭터를 수정해 달라고 하면 `edit` 도구로 해당 부분만 변경합니다.

## 슬래시 커맨드

입력창에 `/skill-name` 형식으로 입력하면 해당 스킬이 사용자 메시지로 활성화됩니다.

예를 들어 `/build-renderer`를 입력하면, `build-renderer` 스킬의 전체 내용이 사용자 메시지에 포함되어 에이전트에게 전달됩니다. 이를 통해 복잡한 절차를 스킬로 정의해두고 간편하게 호출할 수 있습니다.

## 렌더러 액션과의 연동

렌더링된 출력 화면에서 인터랙티브 요소를 클릭하여 에이전트와 상호작용할 수 있습니다.

- **`data-action="send"`** — 해당 요소를 클릭하면 `data-text`에 지정된 텍스트(없으면 요소의 텍스트)가 자동으로 에이전트에게 전송됩니다. 스트리밍 중에는 무시됩니다.
- **`data-action="fill"`** — 해당 요소를 클릭하면 입력창에 텍스트만 채웁니다. 자동 전송하지 않으므로 내용을 확인하거나 수정한 후 직접 전송할 수 있습니다.

이 기능은 Interactive Chat 템플릿의 선택지 시스템 등에서 활용됩니다.
