## 프로젝트란?

프로젝트는 특정 창작 작업을 위한 독립된 작업 공간입니다. 캐릭터 채팅, 소설 집필 등 각 프로젝트가 자체 설정과 파일을 갖고 있으며, 서로 완전히 분리되어 있습니다.

## 프로젝트 폴더 구조

각 프로젝트는 다음과 같은 폴더 구조를 가집니다.

```
{project-name}/
├── _project.json      # 프로젝트 메타데이터 (이름, 생성일)
├── SYSTEM.md           # 시스템 프롬프트 (에이전트 행동 정의)
├── renderer.ts         # 렌더러 (파일→HTML 변환기)
├── skills/             # 스킬 (온디맨드 절차)
├── files/              # 작업 파일 (에이전트의 작업 공간)
│   ├── characters/     # 캐릭터 정의
│   ├── scenes/         # 장면 출력
│   └── world/          # 세계관
└── conversations/      # 대화 기록 (JSONL)
```

## 2-Concept 아키텍처

agentchan의 프로젝트 설계는 핵심 개념 두 가지와 자유로운 파일 공간으로 구성됩니다.

- **SYSTEM.md** — 에이전트 행동의 단일 원천입니다. 작성한 내용이 시스템 프롬프트에 원문 그대로 주입됩니다.
- **Skills** — 온디맨드 절차입니다. 스킬 목록(이름과 설명)만 시스템 프롬프트에 포함되고, 스킬 본문은 에이전트가 활성화할 때만 전달됩니다.
- **files/** — 나머지 전부입니다. 자유로운 구조를 가지며, 시스템이 강제하는 이름이나 디렉토리 구조가 없습니다.

## files/ 디렉토리

`files/`는 에이전트의 작업 공간입니다. 에이전트가 생성하고 수정하는 모든 콘텐츠 파일이 여기에 위치합니다.

- 디렉토리 구조는 완전히 자유이며, 시스템이 특정 구조를 강제하지 않습니다.
- 템플릿이 `characters/`, `scenes/`, `world/` 등 초기 구조를 제안하지만, 자유롭게 변경할 수 있습니다.
- 렌더러가 이 파일들을 읽고 해석하여 왼쪽 패널에 HTML로 시각화합니다.
